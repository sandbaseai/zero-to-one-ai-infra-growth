## Beginner View

Distribution is not the same as posting links everywhere.

For a developer infrastructure product, the better question is:

```text
Where would a serious developer expect to discover or verify this?
```

That is why Week 2 focuses on fewer, better placements.

## Channel Filter

Before submitting anywhere, each channel should pass three questions:

1. Will a real developer trust this place?
2. Can SandBase be presented as infrastructure, not a generic AI tool?
3. Is the submission useful even if it does not immediately drive traffic?

Channels that fail this test can wait.

## Distribution Assets Prepared First

Before submitting SandBase anywhere, the public surface already had:

- a crawlable website
- a technical blog
- a GitHub organization
- a public resource repo
- a LinkedIn company page
- an X account with build notes
- a Discord community entry point
- a consistent agent infrastructure positioning

That order matters.

Directories and communities are not a substitute for trust assets. They amplify what already exists.

## How Codex Helps

Codex is useful in this phase because distribution work creates many small judgment calls:

- whether a directory is credible
- which category fits SandBase
- whether copy sounds like infrastructure or generic AI tooling
- whether a post should include a link or just contribute to the discussion
- whether a channel should wait until the asset is stronger

The operating rule:

```text
Codex can research, draft, compare, and record.
Human confirms public submissions and posts.
```

## What Not To Do

- do not buy backlinks
- do not mass-submit low-quality directories
- do not post the same copy everywhere
- do not force SandBase into generic AI tool categories
- do not launch Product Hunt before the launch assets are ready
