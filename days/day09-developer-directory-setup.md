## Workstream 1 — Daily Operations

Daily operations should stay lightweight:

- LinkedIn: one company or founder-side update if there is something real to say.
- X: one build note or thoughtful reply around agents, runtime, sandboxing, tools, or distributed compute.
- Discord: one question, update, or small prompt that invites builders to respond.

The goal is consistency, not volume.

## Workstream 2 — External Directory Preparation

Directory work should remain careful:

- shortlist credible channels
- prepare technical descriptions
- choose categories that match infrastructure
- avoid broad AI tool directories unless the positioning can stay clear
- record blockers and review status

Submissions can wait if the market/registry concept changes the public story.

## Workstream 3 — Emerging Agent Infrastructure Market

The market idea should be treated as a strategic asset.

Working name:

```text
Emerging Agent Infrastructure
```

Purpose:

```text
Track new, future-facing, technically valuable projects around AI agents that could one day integrate with SandBase.
```

This is not a marketplace for generic AI apps.

The first version should focus on projects with SandBase integration potential:

- agent runtimes
- tool and MCP infrastructure
- sandboxes and computer use
- agent memory
- observability and evals
- deployment and compute
- multi-agent systems
- agent workspaces
- vertical agent infrastructure

Each project should be evaluated by:

- why it matters
- future signal
- technical category
- stage: emerging / growing / established
- SandBase integration potential: runtime / sandbox / tooling / MCP / observability / compute / deployment / community

## Lesson

The registry is valuable only if it is opinionated.

The stronger angle is not "list agent tools." It is:

```text
Track emerging agent infrastructure projects that represent future directions and may integrate with SandBase.
```

The directory lesson is similar: do not chase every completion badge.

For a new infrastructure brand, the better early move is:

```text
Use free, credible profiles to build public trust, but skip privacy-heavy or low-ROI verification steps until there is a clear business reason.
```

The social lesson: public social should not sound like an internal operations log.

Use operations as raw material, but publish the stronger product-facing idea:

```text
Production agents need runtime, sandboxed tools, observability, routing, and compute.
The daily operating work exists to make that point visible to the right builders.
```
