# SandBase.ai Three-Site Growth Plan

Date: 2026-06-25

This is the current promotion target plan for SandBase.ai.

Daily execution should follow:

- [SandBase Daily Growth SOP](sandbase-daily-growth-sop.md)

The goal is to use three public surfaces to build trust, discovery, and developer attention for SandBase:

1. GitHub
2. X
3. LinkedIn

The plan uses open-source assets as the entry point, especially `sandbaseai/awesome-agent-runtime`, and routes attention back to SandBase.ai as the product and infrastructure layer.

## North Star

Make SandBase.ai visible and credible as:

```text
The infrastructure layer for developers building production AI agents.
```

Every public action should help developers understand one of these ideas:

- production agents need infrastructure, not just prompts
- safe execution, browser automation, tool access, memory, evals, observability, and deployment are now core agent-stack layers
- SandBase is building and curating practical infrastructure for this ecosystem
- the open-source projects are useful public assets, not thin marketing pages

## Channel Strategy

## GitHub

Purpose:

Build developer trust and community presence.

Primary actions:

- keep `awesome-agent-runtime` useful and current
- add contribution guidelines and project submission templates
- create issues for category cleanup, missing projects, and source verification
- interact with related projects through useful comments, discussions, and pull requests
- notify maintainers when their projects are included

Daily target:

- find 5-10 relevant projects
- leave 3-5 high-quality GitHub comments or discussion replies
- invite 1-3 maintainers to improve their project description
- record useful external links and replies

Example maintainer note:

```text
We included this project in Awesome Agent Runtime under Browser Automation:
https://github.com/sandbaseai/awesome-agent-runtime

Happy to adjust the description if there is a better positioning.
```

Weekly target:

- 1 project update commit
- 1 GitHub Discussion or issue asking for community submissions
- 20-50 useful interactions
- 5-10 maintainer invitations

Success metrics:

- stars
- forks
- PRs
- issues
- backlinks from GitHub discussions, READMEs, and project docs
- maintainers replying or submitting corrections

- Do not leave generic comments.

- Every interaction should be specific, useful, and technically relevant.
- Prefer linking to open-source assets first, then SandBase.ai when the product context is natural.
- Record what was posted, who replied, and what should be followed up.
