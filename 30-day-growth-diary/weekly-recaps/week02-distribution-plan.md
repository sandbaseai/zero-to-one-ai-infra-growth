# Week 2 Plan — Distribution and Daily Operations

Date range: Day 8 to Day 14

Stage: Week 2 — controlled distribution, daily community work, and issue logging

## Goal

Week 1 built the trust foundation. Week 2 starts using that foundation carefully.

The goal is not to submit SandBase everywhere. The goal is to create the first external discovery paths while continuing to learn what developers actually ask, doubt, click, and ignore.

```text
Distribute what is useful.
Record what happens.
Turn questions into better product, docs, and content.
```

## Week 2 Operating Principle

Distribution must follow the product.

SandBase is agent infrastructure, so the strongest growth assets should help developers understand production AI agents, sandboxed runtime, tool access, model routing, distributed compute, and the operational problems behind them.

Every external action should answer at least one question:

1. Does this help a real developer understand SandBase?
2. Does this create a credible public signal?
3. Does this produce feedback, questions, or learning?
4. Does this avoid looking like generic AI tool promotion?

If the answer is no, the action can wait.

## Weekly Outcomes

By the end of Week 2, we want:

- 5-10 credible external mentions, profiles, or links started
- one developer-facing off-site article drafted or published
- daily X, LinkedIn, and Discord operating rhythm
- a small list of real questions, objections, and repeated confusions
- a clearer view of which channels are worth continued effort
- improvements to GitHub, blog, docs, or landing pages based on actual friction

## Channel Priority

### Higher Priority

- GitHub
- Dev.to
- Emerging Agent Infrastructure registry
- Peerlist
- Indie Hackers
- selected AI infrastructure directories
- selected developer communities
- X and LinkedIn posts tied to actual work

### Use Carefully

- Reddit
- Hacker News
- Product Hunt upcoming
- broad AI tool directories

These channels can be useful, but only when the asset is ready and the post contributes something specific.

### Avoid

- paid backlink packages
- mass directory submission
- repeated copy-paste posts
- low-quality AI tool list sites
- launching Product Hunt without launch assets and reply plan

## Strategic Update

Day 9 raised the priority of a new SandBase ecosystem idea:

```text
Emerging Agent Infrastructure
```

This should be treated as more important than ordinary directory submission. The goal is to track new, future-facing, technically valuable agent infrastructure projects that may one day integrate with SandBase.

This supports Week 2 distribution because it gives SandBase a stronger story:

```text
SandBase is not only promoting itself.
SandBase is mapping the agent infrastructure ecosystem it wants to serve.
```
