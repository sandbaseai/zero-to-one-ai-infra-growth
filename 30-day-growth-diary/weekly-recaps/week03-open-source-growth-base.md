# Week 3 Plan — Open-Source Growth Base

Date range: Day 15 to Day 21

Stage: ecosystem building, open-source assets, and technical distribution

## Goal

Week 3 turns SandBase's open-source work into the main growth engine.

The objective is to make the public GitHub ecosystem useful enough that developers can discover, trust, star, and contribute before they ever see a product pitch.

```text
Use open source as the trust layer.
Use useful technical maps as the distribution layer.
Use SandBase.ai as the product destination.
```

## Weekly Outcomes

By the end of Week 3, we want:

- `awesome-agent-runtime` ready for external contributors
- one clear 500-project announcement package
- a maintainer outreach process that is specific and non-spammy
- one technical cluster around execution sandboxes
- one technical cluster around MCP and tool protocols
- one long-form article draft or published post

## Review Checklist

- Did we make the GitHub project easier to contribute to?
- Did we create at least one durable external link or article?
- Did public comments add technical value without forcing SandBase links?
- Did we collect questions that can become better content?
- Did any maintainer reply or correct our positioning?
