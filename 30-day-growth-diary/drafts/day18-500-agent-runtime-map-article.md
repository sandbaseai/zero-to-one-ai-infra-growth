## Working Title

We Mapped 500 AI Agent Infrastructure Projects

## Short Description

AI agents are moving beyond prompt wrappers. The production stack now includes runtimes, sandboxes, browser automation, tool protocols, memory, evals, model gateways, observability, and deployment infrastructure.

## Article Outline

## 1. Agents Are Becoming Infrastructure

The early agent conversation focused on prompts, tools, and demos. Production systems need more than that.

Once agents touch real files, browsers, APIs, credentials, workflows, and customer data, the infrastructure layer becomes the product boundary.

## 2. The 10-Layer Agent Infrastructure Stack

Use the 10 categories from Awesome Agent Runtime:

- Agent runtime
- Execution sandbox
- Browser automation
- Tool protocol
- App integrations
- Memory/context
- Safety/evals
- Model gateway
- Observability
- Deployment/compute

## 3. Why This Map Exists

This is not a generic AI tools directory.

The goal is to track infrastructure that helps builders run agents in real products:

- control state and workflows
- run code safely
- automate browsers
- connect tools and apps
- store memory and context
- evaluate behavior
- route models
- observe failures
- deploy and scale workloads

## 4. What We Learned From The First 500 Projects

Draft points:

- agent frameworks are maturing quickly, but runtime safety is still uneven
- execution sandboxes and browser automation are becoming first-class agent primitives
- MCP and tool protocols are giving the ecosystem shared language
- observability is moving from prompt logs to traces, evals, costs, and runtime events
- deployment is splitting between model inference and sandbox/tool execution

## 5. How To Use The Repository

Builders can use it to:

- compare infrastructure categories
- find projects to integrate
- identify missing layers in their agent stack
- submit corrections or missing projects
- discover maintainers and adjacent ecosystems
