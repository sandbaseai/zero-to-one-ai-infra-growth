# 外部目录提交素材包

状态：可用于第一次可控提交

负责人：SandBase growth

## 目的

这个素材包用于保证外部目录提交时口径一致。

目标不是把 SandBase 提交到所有目录，而是发布少量可信 profile，强化 SandBase 是面向 production AI agents 的开发者基础设施平台。

## 提交原则

```text
一个可信 profile，比十个泛 AI 工具列表更有价值。
```

提交前先检查：

1. 这个目录是否有真实开发者或 startup 受众？
2. SandBase 是否能被放进 infrastructure、developer tools 或 AI infrastructure 分类？
3. profile 是否能链接官网、GitHub、Docs、Blog、Discord 或 LinkedIn？
4. 这个页面是否像长期可信资产，而不是垃圾目录？
5. 文案是否避免把 SandBase 写成普通 AI app？

如果答案是否定的，先跳过。

## 推荐第一个发布目标

### 首选：Peerlist Project

原因：

- 开发者和 builder 受众更强
- project profile 比普通工具目录更可信
- 适合早期 infrastructure 产品
- 可以展示官网、GitHub 和清晰定位

提交角度：

```text
SandBase is agent infrastructure for developers building production AI agents.
```

### 备选：DevHunt

原因：

- 更偏开发者工具
- 比泛 AI 工具目录更适合
- 可以突出 runtime、sandboxing、tools、models

### 暂缓

- Product Hunt  
  等 launch assets、评论计划、截图和回复节奏准备好再做。

- 泛 AI 目录  
  谨慎使用。和普通 AI app 放在一起会稀释 SandBase 定位。

## 长描述

```text
SandBase is an agent infrastructure platform for developers building production AI agents.

It focuses on the runtime layer behind agent applications: sandboxed execution, safe tool access, model routing, APIs, and distributed compute for agent workloads.

The goal is to help agent builders move from demos to reliable systems that can run tools, use compute, and operate with clearer boundaries.
```

## 第一次提交检查清单

提交前确认：

- 产品名正确
- 分类不会让 SandBase 看起来像普通 AI app
- 一句话介绍包含 `agent infrastructure`
