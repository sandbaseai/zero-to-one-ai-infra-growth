# 第 2 周每日运营 SOP

阶段：可控分发和社区学习

第 2 周每天按这个 SOP 跑一遍。目标不是制造声音，而是让 SandBase 出现在开发者本来就在讨论 agents、runtime、tools、sandboxing、distributed compute、production reliability 的地方。

## 每日结果

每天结束时，我们至少应该完成：

- 1 条产品视角公开更新或动态
- 2-5 次有价值互动，覆盖 X、Discord、Peerlist、LinkedIn 或开发者社区
- 推进 1 个外部 profile、目录、内容资产或生态资产
- 记录 1 个问题、异议或 insight
- 更新当天 build-in-public 实战笔记

## 每日 Checklist

### 0. 渠道巡检

时间盒：10-15 分钟

检查 SandBase 已经运营的渠道：

- X
  - notifications
  - mentions
  - 最近动态的回复
  - 2-3 个高质量主题搜索
- LinkedIn
  - 公司页通知
  - 最近动态评论
  - 是否需要发公司页更新
- Discord
  - `#agent-runtime`
  - `#builder-chat`
  - 是否有新回复或新介绍
- Peerlist
  - notifications
  - SandBase profile / posts 评论
  - 相关 Scroll 讨论
- DevHunt
  - account / profile 状态
  - tool draft 状态
  - 是否出现免费 listing path 或 pricing 变化

不要强行制造动作。如果某个渠道没有有价值互动，就记录“已检查，无动作”。

### 1. 检查搜索和网站健康

时间盒：10 分钟

- 检查 Google Search Console：
  - sitemap 状态
  - 新收录页面
  - crawl errors
  - 异常搜索展示
- 检查最近上线页面是否进入 sitemap：
  - Agent Ecosystem
  - Hackathon Support
  - blog articles
  - docs pages
- 如果发现问题，记录成 P0/P1/P2，不要让它打断整天运营节奏。

### 2. 发布 1 条产品视角观点

时间盒：15 分钟

选择一个渠道：

- X：短技术观点
- LinkedIn：公司可信度和 B2B 信任
- Peerlist：builder 视角更新
- Discord：社区内部讨论

规则：

- 不硬卖
- 不重复贴链接
- 不写泛 AI hype
- 只表达一个清晰技术观点
- 观点要连接 production agents、runtime、tools、sandboxing、model routing、distributed compute 或 reliability

推荐结构：

```text
观察

为什么重要

具体 infrastructure 角度
```

### 3. 做真实社区互动

时间盒：20-30 分钟

互动数量：

- 最少：2 次有价值互动
- 常规：3-5 次有价值互动
- 上限：一旦开始勉强，就停止

优先互动对象：

- 正在展示 agent 项目的 builders
- agent runtime 讨论
- tool calling failure
- sandboxing、browser、shell、file access
- distributed compute for agents
- observability、evaluation、reliability
- emerging agent infrastructure projects

避开：

- AI 赚钱帖
- prompt 课程帖
- roadmap bait
- 没有实质内容的泛 “what are you building”
- 复制粘贴式回复

互动标准：

```text
具体上下文 -> 工程观察 -> 有帮助的问题或 proof point
```

### 4. 推进 1 个分发资产

时间盒：20-40 分钟

每天选择一个：

- 完成一个目录 / profile 草稿
- 准备一个 launch asset
- 优化一个 GitHub README
- 给 Emerging Agent Infrastructure registry 加一个有价值项目
- 准备一篇站外技术文章 outline
- 根据真实反馈优化 docs / blog / landing page

超过预算上限的付费 launch 不当天拍脑袋决定。

### 5. 记录学习

时间盒：10 分钟

更新当天笔记：

- 发了什么
- 在哪里互动
- 链接
- 用户问了什么、忽略了什么
- 阻塞
- 一个经验
- 明天下一步

当天笔记要做到一个月后再看也有用。

