## 给小白的话

分发不是到处贴链接。

对于开发者基础设施产品，更好的问题是：

```text
一个认真开发者，会在哪里发现或验证这个产品？
```

所以第 2 周不是追求数量，而是少一点、准一点、可信一点。

## 目标

启动高质量外部分发：

- 开发者目录提交
- Dev.to 技术文章
- Reddit / Hacker News 参与
- X / LinkedIn / Discord 日常运营
- 记录用户问题和阻塞点

## 渠道筛选原则

每个渠道先问三个问题：

1. 真实开发者会信任这个地方吗？
2. SandBase 能不能以 infrastructure 形象出现，而不是泛 AI 工具？
3. 即使短期没有流量，这个 profile 或链接是否仍然有价值？

不符合就先不做。

## 先准备好的分发资产

在提交任何目录之前，SandBase 已经有：

- 可抓取的官网
- 技术 Blog
- GitHub organization
- 公开 resource repo
- LinkedIn 公司主页
- X build notes
- Discord 社区入口
- 一致的 agent infrastructure 定位

这个顺序很重要。

目录和社区不是可信度本身，它们只是放大已有的可信资产。

## Codex 怎么帮忙

分发阶段有很多小判断：

- 这个目录是否可信
- SandBase 应该放在哪个分类
- 文案听起来像 infrastructure，还是泛 AI 工具
- 这次参与应该带链接，还是只贡献观点
- 某个渠道是否应该等资产更完整再提交

原则：

```text
Codex 可以研究、起草、比较和记录。
公开提交和发帖仍然由人确认。
```

## 不做什么

- 不买外链
- 不群发低质量目录
- 不每个平台复制同一段文案
- 不把 SandBase 强塞进泛 AI 工具分类
- Product Hunt 未准备好前不仓促 launch
