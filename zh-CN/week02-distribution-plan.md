# 第 2 周计划 — 分发和日常运营

时间范围：Day 8 到 Day 14

阶段：第 2 周 — 可控分发、日常社区运营、问题记录

## 目标

第 1 周已经搭好了可信基础盘。第 2 周开始谨慎使用这个基础盘。

目标不是把 SandBase 提交到所有地方，而是建立第一批外部发现路径，同时记录开发者真实会问什么、怀疑什么、点击什么、忽略什么。

```text
分发有用的资产。
记录真实发生的事情。
把问题转成产品、文档和内容的改进。
```

## 第 2 周运营原则

分发必须围绕产品展开。

SandBase 是 agent infrastructure，所以最强的增长资产应该帮助开发者理解 production AI agents、sandboxed runtime、tool access、model routing、distributed compute，以及背后的工程问题。

每个外部动作都要回答至少一个问题：

1. 这是否帮助真实开发者理解 SandBase？
2. 这是否创造了可信的公开信号？
3. 这是否能带来反馈、问题或学习？
4. 这是否避免看起来像普通 AI 工具推广？

如果答案是否定的，这个动作可以先不做。

## 本周结果

第 2 周结束时，我们希望得到：

- 启动 5-10 个可信外部提及、profile 或链接
- 起草或发布 1 篇站外开发者文章
- 建立 X、LinkedIn、Discord 的每日运营节奏
- 记录一批真实问题、异议和重复困惑
- 判断哪些渠道值得继续投入
- 根据真实摩擦改进 GitHub、Blog、Docs 或落地页

## 渠道优先级

### 高优先级

- GitHub
- Dev.to
- Emerging Agent Infrastructure registry
- Peerlist
- Indie Hackers
- 精选 AI infrastructure 目录
- 精选开发者社区
- 围绕真实工作的 X 和 LinkedIn 内容

### 谨慎使用

- Reddit
- Hacker News
- Product Hunt upcoming
- 泛 AI 工具目录

这些渠道可以有用，但前提是资产准备好，并且内容对讨论本身有贡献。

### 避免

- 付费垃圾外链
- 批量目录提交
- 复制粘贴同一段文案
- 低质量 AI 工具列表站
- 没有 launch 资产和回复计划就上 Product Hunt

## 战略更新

Day 9 提高了一个 SandBase 生态想法的优先级：

```text
Emerging Agent Infrastructure
```

它应该比普通目录提交更重要。目标是追踪新的、代表未来方向的、有技术价值的 agent infrastructure 项目，并判断它们未来是否可以和 SandBase 深度集成。

这会强化第 2 周的分发叙事：

```text
SandBase 不是只在推广自己。
SandBase 在梳理它未来要服务的 agent infrastructure 生态。
```
